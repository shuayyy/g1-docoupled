On-policy:
Uses data generated by the current policy.
Example: PPO.

Off-policy:
Can train on data not generated by the current policy.
Usually this includes old replay-buffer data collected by earlier policies.
Examples: SAC, TD3, DQN.

OnPolicyRunner:
The training loop class that collects fresh rollout data from the current policy,
then runs PPO updates on that data.

Legged Gym config structure:
Most environments use two config classes/filesets:
1. Env config:
   robot, terrain, rewards, observations, control, sim, domain randomization
2. Training config:
   policy network settings, PPO hyperparameters, runner/logging/checkpoints

This is standard in legged_gym-style codebases.

PyTorch JIT profiling:
torch._C._jit_set_profiling_mode(False)
torch._C._jit_set_profiling_executor(False)

These disable some PyTorch JIT profiling optimizations.
JIT profiling watches how code runs at runtime and tries to optimize execution based on observed tensor shapes/types/paths.
In RL codebases this is often disabled for simplicity, lower overhead, or stability.
